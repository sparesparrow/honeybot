Project Overview

SCAIPOT (Scam AI Honeypot) is an advanced multi-platform honeypot system designed to detect, engage, and analyze cryptocurrency scammers across Signal, Telegram, and WhatsApp. Powered by Claude AI with prompt caching and MCP-Prompts integration, SCAIPOT reduces operational costs by 90% while maintaining sophisticated conversational capabilities.

Key Features

Multi-Platform Support

Signal, Telegram, & WhatsApp integration

AI-Powered Conversations

Claude 3.5 with MCP-Prompts

12 Scam Templates

Investment, romance, tech support, & more

90% Cost Reduction

Prompt caching with Redis

Advanced Behaviors

URL clicking, VM honeypots, blockchain monitoring

Development Workflow

Clone Repository

Clone the SCAIPOT repository from GitHub and navigate to the project directory

git clone https://github.com/sparesparrow/scaipot.git

Setup Environment

Create virtual environment and install dependencies

python -m venv venv && source venv/bin/activate && pip install -e .[dev]

Configure Services

Copy .env.example to .env and configure API keys, platform tokens, and Redis connection

cp config/templates/.env.example .env

Run Tests

Execute unit and integration tests to verify setup

pytest tests/ -v --cov=src

Start Development Server

Launch SCAIPOT with Docker Compose for local development

docker-compose up -d

Monitor Logs

Watch honeypot activity and scammer interactions in real-time

docker-compose logs -f scaipot

Make Changes

Modify code, add new behaviors, or update honeypot categories

Edit files in src/ and config/

Commit & Push

Follow conventional commits and push to GitHub

git add . && git commit -m "feat: add new behavior" && git push

Deployment Options

🏠 Local Development

docker-compose up -d

Start all services locally

🧪 Staging Environment

docker-compose -f docker-compose.staging.yml up -d

Deploy to staging server

🚀 Production Deployment

./scripts/deploy.sh production

Deploy to production with health checks

Environment Variables

  • ANTHROPIC_API_KEY - Claude API key
  • REDIS_URL - Redis connection string
  • SIGNAL_TOKEN - Signal bot token
  • TELEGRAM_TOKEN - Telegram bot token
  • WHATSAPP_TOKEN - WhatsApp API token
  • MCP_SERVER_URL - MCP-Prompts server URL
  • ADMIN_WEBHOOK - Alert webhook for admin notifications

Security Considerations

  • Never commit API keys or tokens to Git
  • Use environment-specific .env files
  • Enable TLS/SSL for all external connections
  • Regularly update dependencies
  • Monitor logs for suspicious activity
